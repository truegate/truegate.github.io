计算

Amazon EC2

安全并且可以调整大小的云计算容量

Amazon Elastic Compute Cloud 是一种 Web 服务，可以在云中提供安全并且可以调整大小的计算容量。

突出特点

  • 弹性 WEB 规模级计算：客户可以在几分钟内增、减容量；可以同时管理数百个甚至数千个服务器实例；
  • 完全被客户控制的实例
  • 灵活：有多种实例类型、操作系统、软件包可供选择
  • 集成：可与大多数 AWS 服务集成，例如 S3、RDS、VPC，针对多种应用程序提供完整而安全的计算、查询处理和云存储方案
  • 可靠&安全

  • 便宜：EC2 Spot 竞价型实例，使用户能够请求未使用的 EC2 实例，显著降低成本（宣传最大可以节省90%的计算成本）。

    竞价型实例和按需实例的主要差别在于：只有在具有可用的容量时，才能立即启动竞价型实例，竞价型实例的每小时价格根据需求发生变动。在竞价型实例的价格或可用情况发生变化时，Amazon EC2 可能会中断单个竞价型实例。可以采取的一种策略是启动一组核心按需实例，以便为应用程序维护最低级别的保障计算资源，再适机通过竞价型实例来进行补充。另一项策略是，启动具有要求的时长的竞价型实例 (也被称为竞价型限制)，这些实例不会因现货价格更改而中断。
竞价型实例非常适合各种容错且灵活应用程序，例如：测试和开发环境、视频转码、图像渲染、数据分析、批处理作业、后台处理和可选的任务。

  • 轻松入门：可以使用 Console、CLI、SDK进行开发管理

Amazon EC2 Auto Scaling

添加或删除计算容量，应对需求变化。

客户可以定义一些条件，使得系统能够在维持基本可用性的同时，自动动态扩展或缩减 EC2 容量。EC2 Auto Scaling 用于 EC2 实例的队列管理，可以帮助维持队列的正常运行和可用性，确保当前运行的是所需 EC2 实例数量；EC2 Auto Scaling 用于 EC2 实例的动态扩展，实现弹性计算。

突出特点

  • 改进容错能力：能够检测到运行状况不佳的实例，终止并替换为新实例
  • 提高应用程序可用性：确保客户的应用程序始终具备合适的计算容量
  • 降低成本：在必要时增加实例，不必要时减少实例

Amazon Elastic Container Service (Amazon ECS)

在生产环境中运行容器化应用程序

Amazon ECS 是一种高度可扩展的高性能容器编排服务，支持 Docker 容器，客户可以在 AWS 上轻松运行和扩展容器化应用程序。

突出特点

  • 不需要管理基础设施：ECS 支持 AWS Fargate，利用 AWS Fargate 技术，无需选择 Amazon EC2 实例类型、无需预置和扩展运行容器的虚拟机集群，也不再需要安排要在集群上运行的容器并保持其可用性。Fargate 让您可以专注于设计和运行应用程序，无需处理底层基础设施。
  • 将各种内容容器化：通过 ECS 可以轻松构建各种类型的容器化应用程序，包括了常驻应用程序、微服务、批处理任务和机器学习应用程序。您可以将传统的 Linux 或 Windows 应用程序从本地迁移到云中，然后使用 ECS 将其作为容器化应用程序运行。
  • 安全：ECS 在客户自己的 Amazon VPC 中启动容器，单独的 VPC 安全组和网络 ACL 不与其他客户共享计算资源。还可以使用 IAM 为每个容器分配细粒度的访问权限，以便限制对各项服务的访问，控制某个容器可以访问哪些资源。
  • 规模效应：几秒钟就能启动数十个或者数万个 Docker 容器，同时不增加额外的复杂性。
  • 与其他 AWS 服务配合使用：与以下 AWS 服务集成，为客户提供可以运行多种容器化应用程序或服务的完整解决方案：Elastic Load Balancing、Amazon VPC、AWS IAM、Amazon ECR、AWS Batch、Amazon CloudWatch、AWS CloudFormation、AWS CodeStar 和 AWS CloudTrail。

典型应用场景

  • 微服务
  • 批处理
  • 将应用程序迁移到云
  • 机器学习

Amazon Elastic Container Registry （Amazon ECR)

存储和检索 Docker 镜像

ECR 是完全托管的 Docker 容器注册表，可使开发人员轻松存储、管理和部署 Docker 容器映像。Amazon ECR 与 Amazon ECS 集成，大大简化生产工作流程的开发。ECR 没有预付费用或长期合约，只需为存储库中存储的数据量以及传输到 Internet 的数据量付费。

突出特点

  • 完全托管并且高度可用：不再需要为支持容器注册表而运营和扩展基础设施。无需安装和管理任何软件，也无需扩展基础设施。只需将您的容器映像放入 Amazon ECR，然后在需要部署时再取回该映像即可。 ECR 具有冗余、持久且高度可扩展的架构。您的容器映像高度可用且可访问，这使您能够为应用程序可靠部署新的容器。

Amazon Elastic Container Service for Kubernetes (Amazon EKS)

高度可用并可以扩展的 K8S 服务

Amazon EKS 可在三个可用区中运行三个 Kubernetes 主节点，从而确保了高可用性。Amazon EKS 可以自动检测和替换运行状况不佳的主节点，并对主节点进行自动版本升级和修补。

突出特点

  • 完全托管并且高度可用：K8S 由工作程序节点和主节点两部分组成。工作程序节点是运行容器的服务器，而主节点控制工作程序节点的群集，并对容器进行规划并将其放置在特定的服务器上。Amazon EKS 可以针对每个群集自动运行和管理三个 K8S 主节点，让客户可以轻松运行高度可用的 K8S 群集。Amazon EKS 管理的多可用区架构可以在一个 AWS 可用区丢失时保证弹性。Amazon EKS 还可以自动检测和替换运行状况不佳的主节点，并对主节点进行自动版本升级和修补。

典型应用场景

  • 微服务
  • 批处理
  • 混合容器部署
  • 机器学习

AWS Batch

任意规模的完全托管式批处理。简单来说，Batch 是可以预定义执行任务所需的计算机资源，然后在任务触发时能够自动创建对应的实例。

AWS Batch 可根据提交的批处理任务的数量和特定资源要求，动态预置计算资源 (CPU 或 内存优化型实例) 的最佳数量和类型。借助 AWS Batch，无需安装和管理运行您的任务所使用的批处理计算软件或服务器群集，从而使您能够专注于分析结果和解决问题。AWS Batch 可以跨多种 AWS 计算服务和功能 (如 Amazon EC2 和竞价型实例) 计划、安排和执行您的批处理计算工作负载。 AWS Batch 不收取任何额外费用。只需为存储和运行您的批处理任务而创建的 AWS 资源 (如 EC2 实例) 付费。

突出特点

  • 完全托管：AWS Batch 可为客户管理所有基础设施，从而避免了预置、管理、监控和扩展批处理计算任务所带来的复杂性。
  • 与 AWS 集成
  • 成本优化的资源预置

典型应用场景

  • 金融服务-交易后分析：自动分析每天的交易费用、执行报告和市场绩效
  • 数字媒体-视觉效果呈现：自动处理内容呈现工作负载，并通过执行依赖关系或资源排程降低人为干预的必要性。

AWS Elastic Beanstalk

运行和管理 Web 应用 在熟悉的服务器（例如 Apache 、Nginx、Passenger 和 IIS）上部署和扩展使用 Java、.NET、PHP、Node.js、Python、Ruby、GO 和 Docker 开发的 Web 应用程序和服务。

突出特点

  • 入门快速，使用简单：客户只需上传代码，Elastic Beanstalk 将自动处理容量预配置、负载平衡、自动扩展及应用程序运行状况监控等部署详细信息。几分钟内，您的应用程序即可使用，无需您处理任何基础设施或资源配置工作。
  • 开发人员生产力：客户只需集中精力编写代码，其他管理和配置服务器、数据库、加载平衡器、防火墙和网络等工作交由 Elastic Beanstalk 代劳。
  • 无限扩展：Elastic Beanstalk 会根据应用程序的具体需求使用可以轻松调整的 Auto Scaling 设置自动对其进行扩缩。
  • 完善的资源控制： 使用Elastic Beanstalk，可以实施对 AWS 资源的“彻底检查”，并保留对应用程序所需 AWS 资源的完全控制。

AWS Fargate

无需管理服务器或集群即可运行容器

AWS Fargate 是一项适用于 Amazon ECS 和 EKS 的技术，让您无需管理服务器或集群即可运行容器。使用 AWS Fargate，您不必再预置、配置和扩展虚拟机集群即可运行容器，客户可以专注于设计和构建应用程序，而不是管理运行应用程序的基础设施。

突出特点

  • 无需管理集群
  • 无缝扩展
  • 与 ECS/EKS 集成

AWS Lambda

运行代码，无需顾虑服务器，只需按使用时间付费

借助 Lambda，您几乎可以为任何类型的应用程序或后端服务运行代码，而且全部无需管理。只需上传您的代码，Lambda 会处理运行和扩展高可用性代码所需的一切工作。您可以将您的代码设置为自动从其他 AWS 服务触发，或者直接从任何 Web 或移动应用程序调用。

突出特点

  • 无需管理服务器
  • 持续扩展
  • 次秒级计量：使用 AWS Lambda 时，会按代码执行时间 (以每 100 毫秒为单位) 和代码触发次数收费。代码未运行时，无需支付任何费用。

典型应用场景

  • 数据处理：实时文件处理、实时数据流处理
  • 后端：IoT后端、移动后端、WEB 应用程序

AWS Serverless Application Repository

发现、部署和发布无服务器应用程序

借助 AWS Serverless Application Repository，您能够轻松针对常见使用案例 (如 Web 和移动后端、事件和数据处理、日志记录、监控和 IoT 等) 部署示例代码、组件和完整应用程序。每个应用程序均使用 AWS 无服务器应用程序模型 (SAM) 模板打包，该模板定义了所使用的 AWS 资源。公开分享的应用程序还包括指向应用程序源代码的链接。使用 Serverless Application Repository 不会产生额外费用，您只需为您部署的应用程序中使用的 AWS 资源付费。

VMware Cloud on AWS

将 VMware 环境迁移和扩展到 AWS 云 借助 VMware Cloud on AWS，组织可以在本地数据中心和 AWS 云中使用相同的 VMware Cloud Foundation 技术（包括 vSphere、vSAN、NSX 和 vCenter Server），无需购买任何新硬件或自定义硬件、重写应用程序或修改操作模式，从而实现对混合 IT 操作的简化。

Amazon Elastic File System(EFS)

可靠且可扩展的弹性文件存储

典型应用场景

EFS 是一项区域性服务，专为实现高可用性和耐久性而设计，支持各种使用案例，包括

  • Web 服务和内容管理
  • 企业应用程序
  • 媒体和娱乐处理工作流
  • 主目录
  • 数据库备份
  • 开发人员工具
  • 容器存储
  • 大数据分析
云存储有三种类型：对象存储、文件存储和数据块存储。每种类型均有独特的优势。

1. 对象存储 – 在云中开发的应用程序通常可利用对象存储的高度可扩展性和元数据特性。Amazon Simple Storage Service (Amazon S3) 等对象存储解决方案非常适合用于从头构建需要扩展和灵活性的现代应用程序；您还可以使用这些解决方案导入现有数据存储以进行分析、备份或存档。
2. 文件存储 – 许多应用程序需要访问共享文件并需要文件系统。通常使用网络附加存储 (NAS) 服务器支持这种类型的存储。Amazon Elastic File System (EFS) 等文件存储解决方案非常适合大型内容存储库、开发环境、媒体存储或用户主目录等使用案例。
3. 数据块存储 – 数据库或 ERP 系统等其他企业应用程序通常需要针对每个主机的专用低延迟存储。这种存储与直接连接存储 (DAS) 或存储区域网络 (SAN) 类似。Amazon Elastic Block Store (EBS) 等基于数据块的云存储解决方案使用各个虚拟服务器进行预配置，可提供高性能工作负载所需的超低延迟。

数据库

Amazon Aurora

兼容 MySQL 和 PostgreSQL 的关系数据库，既具有高端商用数据库的性能和可用性，又具有开源数据库的简单性和成本效益

突出特点

Aurora 的速度最高可以达到标准 MySQL 数据库的五倍、标准 PostgreSQL 数据库的三倍。它可以实现商用数据库的安全性、可用性和可靠性，而成本只有商用数据库的 1/10。

Amazon Relational Database Service(RDS)

提供六种常用数据库（包括 Amazon Aurora、PostgreSQL、MySQL、MariaDB、Oracle 和 Microsoft SQL Server）引擎的托管关系数据库

DynamoDB

快速灵活的 NoSQL 数据库服务，适合所有需要一致性且延迟低于 10 毫秒的任意规模的应用程序。

Amazon ElastiCache

在云中完全托管的内存数据存储和缓存。与 Redis 和 Memcached 兼容。实现亚毫秒级延迟，支持实时应用程序。

突出特点

ElastiCache 支持两种开源的内存引擎：

  • Redis：适用于 Redis 的 ElastiCache 具有完全托管、可扩展和安全等特点，是 Web 应用程序、移动应用程序、医疗保健应用程序、金融应用程序、游戏、广告技术和物联网等高性能使用案例的理想之选。
  • Memcached： 一种广泛采用的内存对象缓存系统。ElastiCache 与 Memcached 保持协议兼容，因此您当前用于现有 Memcached 环境的常用工具可以无缝应用于该服务。

    典型应用

    使用 Amazon ElastiCache，可以显著改进各种应用程序工作负载（例如社交网络、游戏、广告技术、媒体共享和 Q&A 门户）或计算密集型工作负载（例如推荐引擎）的延迟和吞吐量。当用作缓存时，ElastiCache 可存储内存中的关键数据，以便实现低延迟访问，从而提高应用程序的性能。此信息可能包括 I/O 密集型数据库查询的结果或计算密集型计算的结果。对于需要快速内存数据结构服务器的应用程序、高度可用的缓存或消息代理而言，Redis 则最为有用。

Redshift

Amazon Redshift 是一种快速且完全托管的数据仓库，让您可以使用标准 SQL 和现有的商业智能 (BI) 工具经济高效地轻松分析您的所有数据。

突出特点

专为数据仓库而优化，对于大小在 100GB 到 1EB 或更高的数据集，拥有非常高的查询性能。对于 PB 级本地数据，它使用列式存储、数据压缩及区域映射，降低了执行查询所需的 I/O 数量。

典型应用场景

迁移

AWS Application Discovery Service(ADS)

ADS 可以收集有关企业客户的本地数据中心的信息，从而帮助企业客户规划迁移项目。

数据中心迁移的规划工作可能会涉及数千个互相高度依赖的负载，确定服务器利用率数据和依赖关系映射是迁移流程中的一个重要前期步骤。ADS 可以收集并呈现服务器的配置、使用和行为数据，帮助您更好地了解您的工作负载。

AWS Database Migration Service(DMS)

在停机时间最短的状态下将本地数据库迁移至 AWS

DMS 支持同构迁移（如从 Oracle 迁移到 Oracle），异构迁移（如从 SQL Server 迁移到 MySQL）。它还支持您从任意受支持的源位置（包括 Amazon Aurora、PostgreSQL、MySQL、MariaDB、Oracle、SAP ASE 和 SQL Server）将数据流式传输到 Amazon Redshift，以便在 PB 级数据仓库中对数据进行整合和轻松分析。DMS 还可用于连续数据复制，且高度可用。

Migration Hub

AWS Migration Hub 提供单个位置来跨多个 AWS 和合作伙伴解决方案跟踪应用程序迁移的进度

通过 Migration Hub，可以选择任意 AWS 或合作伙伴的迁移工具，并可以了解整个应用程序产品组合的迁移状态（如关键指标、进度信息）

AWS Server Migration Service(SMS)

SMS 是一种无代理服务，可以让您更加轻松快速地将成千上万的本地工作负载迁移到 AWS。借助 SMS，您可以自动执行实时服务器卷的增量复制、对其制定计划以及进行追踪，从而能够更轻松地协调大规模服务器迁移

AWS Snowball

Snowball 是一种 PB 级数据传输解决方案，它使用安全设备在 AWS 云之间传输大量数据。可解决大规模数据传输的难题（包括高昂网络成本、较长传输时间和安全问题）,成本可低至高速 Internet 费用的五分之一

借助 Snowball，您无需编写任何代码或购买任何硬件即可传输您的数据。只需在 AWS 管理控制台中创建一个任务，然后 Snowball 设备将自动交付于您*。当它到达后，将该设备挂载到您的本地网络、下载并运行 Snowball 客户端来建立连接，然后使用该客户端选择要传输到该设备的文件目录。客户端随后将对文件进行加密，并将其高速传输至该设备。当传输完成后即可返还该设备，E Ink 运送标签将自动更新，您可以通过 Amazon Simple Notification Service (SNS)、短信或直接在控制台中跟踪任务状态。

AWS Snowball Edge

AWS Snowball Edge 是一台容量为 100TB 的数据传输设备，具有板载存储与计算功能。可以实现大量数据的移入和移出 AWS 操作。

可以将其用作大型本地数据集的临时存储层，或者在远程或离线位置支持本地工作负载。

典型应用场景

  • 物联网

Amazon Simple Queue Service (SQS)

适用于微服务、分布式系统和无服务器应用程序的完全托管的消息队列

突出特点

  • 标准队列 Amazon SQS 提供标准队列作为默认队列类型。标准队列使您能够每秒处理近乎无限数量的事务。标准队列可确保每条消息至少被传送一次。但是，由于允许高吞吐量的高度分布式架构，偶尔会有一条消息的某个副本不按顺序传送。标准队列提供最大努力排序，可保证消息大致按其发送的顺序进行传送。
  • FIFO 队列 FIFO 队列是对标准队列的补充。这种队列类型最重要的功能是 FIFO (先进先出) 传送和一次性处理：让消息的发送顺序和接收顺序严格保持一致，且消息只传送一次并保留到用户将其处理和删除；重复的消息不会被引入队列。FIFO 队列还支持消息组，即允许在单个队列中传送多个有序流。FIFO 队列的上限是 300 个事务/秒 (TPS)，但具有标准队列的全部功能。

适用于微服务、分布式系统和无服务器应用程序的发布/订阅消息收发和移动通知

桌面计算服务，利用 Amazon WorkSpaces，您可以为您的用户轻松预置基于云的虚拟 Microsoft Windows 桌面，让用户使用支持的任何设备随时随地访问自己需要的文档、应用程序和资源

Amazon AppStream 2.0

借助 Amazon AppStream 2.0，您可以轻松将现有桌面应用程序导入 AWS 并开始将其流式传输到兼容 HTML5 的浏览器，您的用户可始终访问其应用程序的最新版本

#物联网

AWS IoT Core

AWS IoT Core 是一款托管的云平台，使互联设备可以轻松安全地与云应用程序及其他设备交互。AWS IoT Core 可以支持数十亿台设备和数万亿条消息，并且可以对这些消息进行处理并将其安全可靠地路由至 AWS 终端节点和其他设备。借助 AWS IoT Core，您的应用程序可以随时跟踪所有设备并与其通信，即使这些设备未处于连接状态也不例外。

Amazon FreeRTOS

MCU OS，实现对低功耗的小型边缘设备进行编程、部署、安全保护、连接和管理

AWS Greengrass

互联设备的本地计算、消息收发、数据缓存、同步和 ML Inference 功能

借助 AWS Greengrass，互联设备可以运行 AWS Lambda 函数、同步设备数据以及与其他设备安全通信 – 甚至无需连接互联网

AWS IoT 1-Click

AWS IoT 1-Click 是一种服务，简单设备可以通过该服务轻松触发执行特定操作的 AWS Lambda 函数

AWS IoT Analytics

AWS IoT Analytics 是一项完全托管的服务，让您能够对大量 IoT 数据轻松运行复杂的分析，无需担心构建自己的 IoT 分析平台通常会产生的所有成本和复杂性

AWS IoT Analytics 完全托管并且自动扩展，可以处理高达 PB 级的 IoT 数据。借助 IoT Analytics，您能够分析来自数百万台设备的数据并构建速度快、响应性高的 IoT 应用程序，无需管理硬件或基础设施。

AWS IoT Button

一款基于 Amazon Dash Button 硬件的可编程按钮

AWS IoT Device Defender

AWS IoT Device Defender 会不断审核与设备关联的安全策略，以确保设备始终遵循安全最佳实践

AWS IoT Device Management

安全地大规模注册、组织、监控和远程管理 IoT 设备

游戏开发

Amazon GameLift

一款在 AWS 上构建的专用游戏服务器托管和匹配解决方案，使用它，可节省长达数千小时的工程时间，并能降低成本

Amazon Lumberyard

免费、功能强大、完全可以自定义的一款游戏引擎
