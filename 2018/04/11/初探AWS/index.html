发表于 2018-04-11 | 分类于 技术 |

计算

Amazon EC2

安全并且可以调整大小的云计算容量

Amazon Elastic Compute Cloud 是一种 Web 服务，可以在云中提供安全并且可以调整大小的计算容量。

突出特点

  • 弹性 WEB 规模级计算：客户可以在几分钟内增、减容量；可以同时管理数百个甚至数千个服务器实例；
  • 完全被客户控制的实例
  • 灵活：有多种实例类型、操作系统、软件包可供选择
  • 集成：可与大多数 AWS 服务集成，例如 S3、RDS、VPC，针对多种应用程序提供完整而安全的计算、查询处理和云存储方案
  • 可靠&安全

  • 便宜：EC2 Spot 竞价型实例，使用户能够请求未使用的 EC2 实例，显著降低成本（宣传最大可以节省90%的计算成本）。

    竞价型实例和按需实例的主要差别在于：只有在具有可用的容量时，才能立即启动竞价型实例，竞价型实例的每小时价格根据需求发生变动。在竞价型实例的价格或可用情况发生变化时，Amazon EC2 可能会中断单个竞价型实例。可以采取的一种策略是启动一组核心按需实例，以便为应用程序维护最低级别的保障计算资源，再适机通过竞价型实例来进行补充。另一项策略是，启动具有要求的时长的竞价型实例 (也被称为竞价型限制)，这些实例不会因现货价格更改而中断。
竞价型实例非常适合各种容错且灵活应用程序，例如：测试和开发环境、视频转码、图像渲染、数据分析、批处理作业、后台处理和可选的任务。

  • 轻松入门：可以使用 Console、CLI、SDK进行开发管理

Amazon EC2 Auto Scaling

添加或删除计算容量，应对需求变化。

客户可以定义一些条件，使得系统能够在维持基本可用性的同时，自动动态扩展或缩减 EC2 容量。EC2 Auto Scaling 用于 EC2 实例的队列管理，可以帮助维持队列的正常运行和可用性，确保当前运行的是所需 EC2 实例数量；EC2 Auto Scaling 用于 EC2 实例的动态扩展，实现弹性计算。

突出特点

  • 改进容错能力：能够检测到运行状况不佳的实例，终止并替换为新实例
  • 提高应用程序可用性：确保客户的应用程序始终具备合适的计算容量
  • 降低成本：在必要时增加实例，不必要时减少实例

Amazon Elastic Container Service (Amazon ECS)

在生产环境中运行容器化应用程序

Amazon ECS 是一种高度可扩展的高性能容器编排服务，支持 Docker 容器，客户可以在 AWS 上轻松运行和扩展容器化应用程序。

突出特点

  • 不需要管理基础设施：ECS 支持 AWS Fargate，利用 AWS Fargate 技术，无需选择 Amazon EC2 实例类型、无需预置和扩展运行容器的虚拟机集群，也不再需要安排要在集群上运行的容器并保持其可用性。Fargate 让您可以专注于设计和运行应用程序，无需处理底层基础设施。
  • 将各种内容容器化：通过 ECS 可以轻松构建各种类型的容器化应用程序，包括了常驻应用程序、微服务、批处理任务和机器学习应用程序。您可以将传统的 Linux 或 Windows 应用程序从本地迁移到云中，然后使用 ECS 将其作为容器化应用程序运行。
  • 安全：ECS 在客户自己的 Amazon VPC 中启动容器，单独的 VPC 安全组和网络 ACL 不与其他客户共享计算资源。还可以使用 IAM 为每个容器分配细粒度的访问权限，以便限制对各项服务的访问，控制某个容器可以访问哪些资源。
  • 规模效应：几秒钟就能启动数十个或者数万个 Docker 容器，同时不增加额外的复杂性。
  • 与其他 AWS 服务配合使用：与以下 AWS 服务集成，为客户提供可以运行多种容器化应用程序或服务的完整解决方案：Elastic Load Balancing、Amazon VPC、AWS IAM、Amazon ECR、AWS Batch、Amazon CloudWatch、AWS CloudFormation、AWS CodeStar 和 AWS CloudTrail。

Amazon Elastic Container Service for Kubernetes (Amazon EKS)

高度可用并可以扩展的 K8S 服务

Kubernetes 是一种开源系统，用于自动化部署、扩展和管理容器化应用程序。Amazon EKS 可在三个可用区中运行三个 Kubernetes 主节点，从而确保了高可用性。Amazon EKS 可以自动检测和替换运行状况不佳的主节点，并对主节点进行自动版本升级和修补。
